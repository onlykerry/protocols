Porting

A quick guide to porting uIP to other architectures:

There is no need to make any changes to the actual TCP/IP code, but a device driver for the target's network device (ethernet controller/serial port/whatever) has to be written. The the actual system integration part (i.e., the main control loop, which should call the uIP functions when data arrives and when the periodical timer expires) also has to be done.

A step-though guide could look like this:

  1. Read the documentation provided above.
  2. Create a new directory for your port under the uip-0.5/ directory (choose a name that gives a short description of the CPU architecture and/or C compiler used).
  3. Copy the uip_arch.[ch] files from the bsd/ subdirectory into the new directory. These are generic C implementations of the checksum algorithm as well as some other "helper" function.
  4. Copy the uipopt.h file from the bsd/ subdirectory.
  5. Edit the uipopt.h file to suit your project (the file should be self-explanatory).
  6. Write a device driver for your particular hardware. (This is probably the hardest part.) Look in bsd/tundev.c and cc65/rs232dev.c for examples of how a device driver can look.
  7. Write the main control loop so that the uIP functions are called when they should be. Look in the bsd/main.c and cc65/main.c for examples of how a main control loop can look.
  8. Write a makefile and compile the code. Be sure to include the ../uip/uip.c file in your project (and the .c files in your subdirectory, of course). If you are using the web server application, also include the files ../apps/httpd/httpd.c ../apps/httpd/cgi.c ../apps/httpd/fs.c.
  9. Find and correct all bugs. (This can also be quite tricky.)
